# Contributing

Thanks for your interest in contributing.

## How to contribute
1. Fork the repo and create a feature branch.
2. Keep changes focused and well-described.
3. If you modify research mechanics, include:
   - Rationale
   - Any measurement or validity implications
   - Example session demonstrating the change
4. Open a Pull Request.

## Style
- Keep this repository **no-build** where possible (single-file HTML).
- Prefer small, readable vanilla JS additions over heavy frameworks.
